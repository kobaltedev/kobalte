# Stepper

A multi-step interface that guides users through a sequence of related tasks or content. Built on top of the Tabs component for accessibility.

## Import

```ts
import { Stepper } from "@kobalte/core/stepper";
// or
import { Root, Item, ... } from "@kobalte/core/stepper";
```

## Features

- Full keyboard navigation
- Controlled and uncontrolled modes
- Support for validation between steps
- Customizable step indicators and content
- Support for completed state
- Built on top of Tabs component for accessibility
- Can be controlled or uncontrolled

## Anatomy

The stepper consists of:

- **Stepper:** The root container for the stepper component.
- **Stepper.List:** Contains the list of step indicators.
- **Stepper.Item:** An individual step indicator.
- **Stepper.Trigger:** The button that allows users to navigate to a specific step.
- **Stepper.Separator:** Visual separator between step indicators.
- **Stepper.Content:** Content associated with each step.
- **Stepper.CompletedContent:** Content shown when all steps are completed.
- **Stepper.PrevTrigger:** Button to navigate to the previous step.
- **Stepper.NextTrigger:** Button to navigate to the next step.

```tsx
<Stepper maxSteps={3}>
  <Stepper.List>
    <Stepper.Item index={0}>
      <Stepper.Trigger step={0}>Step 1</Stepper.Trigger>
    </Stepper.Item>
    <Stepper.Separator />
    <Stepper.Item index={1}>
      <Stepper.Trigger step={1}>Step 2</Stepper.Trigger>
    </Stepper.Item>
    <Stepper.Separator />
    <Stepper.Item index={2}>
      <Stepper.Trigger step={2}>Step 3</Stepper.Trigger>
    </Stepper.Item>
  </Stepper.List>

  <Stepper.Content index={0}>Step 1 content</Stepper.Content>
  <Stepper.Content index={1}>Step 2 content</Stepper.Content>
  <Stepper.Content index={2}>Step 3 content</Stepper.Content>

  <Stepper.CompletedContent>All steps completed!</Stepper.CompletedContent>

  <div>
    <Stepper.PrevTrigger>Previous</Stepper.PrevTrigger>
    <Stepper.NextTrigger>Next</Stepper.NextTrigger>
  </div>
</Stepper>
```

## Usage

### Form Integration

The Stepper component can be used to create multi-step forms with validation.

<Preview>
  <FormExample />
</Preview>

```tsx
export const FormExample = () => {
  const [currentStep, setCurrentStep] = createSignal(0);
  const [formData, setFormData] = createSignal({
    name: "",
    email: "",
  });

  return (
    <Stepper
      step={currentStep()}
      onStepChange={setCurrentStep}
      maxSteps={3}
    >
      <Stepper.List>
        <Stepper.Item>
          <Stepper.Trigger>1</Stepper.Trigger>
          <span>Personal</span>
        </Stepper.Item>

        <Stepper.Separator />

        {/* Additional steps */}
      </Stepper.List>

      <form>
        <Stepper.Content index={0}>
          <input
            type="text"
            value={formData().name}
            onInput={(e) => setFormData({ ...formData(), name: e.currentTarget.value })}
          />
        </Stepper.Content>

        {/* Additional step content */}
      </form>
    </Stepper>
  );
};
```

### Validation

Steps can include validation logic before proceeding to the next step.

<Preview>
  <ValidatedExample />
</Preview>

```tsx
export const ValidatedExample = () => {
  const [currentStep, setCurrentStep] = createSignal(0);
  const [errors, setErrors] = createSignal({});

  const validateStep = (step: number) => {
    const newErrors = {};
    // Add validation logic
    setErrors(newErrors);
    return Object.keys(newErrors).length === 0;
  };

  const handleStepChange = (newStep: number) => {
    if (newStep > currentStep()) {
      if (validateStep(currentStep())) {
        setCurrentStep(newStep);
      }
    } else {
      setCurrentStep(newStep);
    }
  };

  return (
    <Stepper
      step={currentStep()}
      onStepChange={handleStepChange}
      maxSteps={3}
    >
      {/* Stepper content */}
    </Stepper>
  );
};
```

## API Reference

### Stepper (Root)

The root component that wraps all stepper elements.

| Prop | Type | Description |
|:--|:--|:--|
| `step` | `number` | The controlled step index |
| `defaultStep` | `number` | The default step index when initially rendered |
| `maxSteps` | `number` | The total number of steps (required) |
| `onStepChange` | `(step: number) => void` | Event handler called when the step changes |
| `disabled` | `boolean` | When true, prevents interaction with the stepper |

### Stepper.Item

| Prop | Type | Description |
|:--|:--|:--|
| `index` | `number` | The zero-based index of this step (required) |

### Stepper.Trigger

| Prop | Type | Description |
|:--|:--|:--|
| `step` | `number` | The step index this trigger navigates to (required) |

### Stepper.Content

| Prop | Type | Description |
|:--|:--|:--|
| `index` | `number` | The step index this content belongs to (required) |

## Data Attributes

| Component | Attribute | Description |
|:--|:--|:--|
| `Stepper.Item` | `data-current` | Present when this is the current step |
| `Stepper.Item` | `data-complete` | Present when this step is completed |
| `Stepper.Trigger` | `data-disabled` | Present when the trigger is disabled |
| `Stepper.CompletedContent` | `data-completed` | Present when all steps are completed |

## Keyboard Interactions

| Key | Description |
|:--|:--|
| <Kbd>Tab</Kbd> | Moves focus to the next focusable element |
| <Kbd>Space</Kbd> or <Kbd>Enter</Kbd> | Activates the focused step |
| <Kbd>ArrowLeft</Kbd> | Moves to the previous step |
| <Kbd>ArrowRight</Kbd> | Moves to the next step |

## Accessibility

The Stepper component is built on top of the Tabs component and follows the [WAI-ARIA Tabs Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/). It includes proper ARIA attributes and keyboard navigation support.

- Steps are automatically labeled with their position (e.g., "Step 1 of 3")
- Current step is indicated with `aria-current="step"`
- Navigation buttons are properly labeled for screen readers
- Completed steps are announced to screen readers
